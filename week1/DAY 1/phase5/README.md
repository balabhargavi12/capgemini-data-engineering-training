# Phase 5 – Final Data Engineering Pipeline

## 📌Problem Understanding
In this phase, the goal is to build a complete data engineering pipeline to process and analyze data. The objective is to generate meaningful business insights such as sales trends, customer behavior, and revenue distribution. This simulates a real-world scenario where large datasets need to be processed efficiently.

##⚙️ Approach Taken
- Loaded datasets into PySpark environment  
- Cleaned and preprocessed the data  
- Performed joins between multiple datasets  
- Applied transformations and aggregations  
- Generated final reporting tables  

The workflow follows the ETL (Extract Transform Load) process:
- Extract → Load raw data  
- Transform → Clean and process data  
- Load → Store final results  

## 🔧Key Steps / Transformations
- Data cleaning (handling null values and inconsistencies)  
- Filtering relevant records  
- Joining datasets using keys  
- GroupBy and aggregation operations  
- Creating derived columns  
- Generating summary tables for insights  

## 🧠 Learnings and Challenges

### ✅Learnings

- Implementing efficient transformations  
- Converting business requirements into technical solutions  

### ⚠️Challenges
- Managing joins across multiple datasets  
- Handling missing and inconsistent data  
- Writing correct aggregation logic  
- Debugging PySpark transformations
